# 安装指南

## 方法一：直接安装（推荐）

1. 下载插件压缩包或克隆仓库
2. 将整个 `encode_helper` 文件夹复制到 Mulby 的插件目录：
   - **Windows**: `%APPDATA%\Mulby\plugins\`
   - **macOS**: `~/Library/Application Support/Mulby/plugins/`
   - **Linux**: `~/.config/Mulby/plugins/`

3. 重启 Mulby 或进入插件管理界面重新加载插件
4. 在插件管理中找到 "编码小助手" 并启用

## 方法二：开发模式安装

1. 克隆或下载插件源代码
2. 在插件目录中运行：
   ```bash
   npm install
   npm run build
   ```

3. 将构建好的插件文件夹复制到 Mulby 插件目录
4. 启用插件

## 验证安装

1. 打开 Mulby 主界面
2. 输入以下任意关键词测试：
   - `时间转换`
   - `base64`
   - `url编码`
   - `md5`
   - `unicode`
   - `uuid`
   - `进制转换`
   - `html转义`

3. 如果插件正常工作，会显示对应的功能界面

## 更新插件

1. 备份当前插件配置（如果需要）
2. 删除旧的插件文件夹
3. 按照安装步骤重新安装新版本
4. 重新启用插件

## 故障排除

### 插件未显示
- 检查插件文件夹是否在正确的目录
- 检查插件文件夹名称是否为 `encode_helper`
- 重启 Mulby 并重新加载插件

### 功能无法使用
- 检查是否已启用插件
- 检查浏览器控制台是否有错误信息
- 确保系统时间和时区设置正确

### 构建失败
- 确保已安装 Node.js 16+
- 运行 `npm install` 安装所有依赖
- 检查网络连接是否正常

## 支持与反馈

如果在使用过程中遇到问题或有改进建议，请：
1. 检查 README.md 中的常见问题
2. 查看控制台错误信息
3. 提交 Issue 到项目仓库

## 系统要求

- **Mulby**: 最新版本
- **操作系统**: Windows 10+, macOS 10.15+, Linux
- **Node.js**: 16+（仅开发需要）
- **内存**: 建议 2GB+
